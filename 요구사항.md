# 요구 사항

## 개요

개인 포트폴리오를 위한 웹 사이트를 구축합니다. 이를 위해 다음과 같은 요구 사항을 충족해야 합니다.
주요 콘텐츠는 제작한 다큐멘터리 소개, 출판한 책 소개, 기사 소개 등입니다.
이 내용들을 블로그 형식의 웹 사이트로 구성하여 방문자들이 쉽게 접근할 수 있도록 합니다.

- [개인 포트폴리오 예시](https://jonghoonahn.com/)

## 주요 요구사항

1. 홈 페이지: 방문자가 첫 번째로 접하는 페이지로, 간단한 자기 소개와 주요 프로젝트 링크, 블로그 섹션의 글들을 포함해야 합니다.
2. 블로그 섹션: 제작한 다큐멘터리, 출판한 책, 기사 등 소개 글을 작성하고 게시할 수 있는 블로그 기능을 포함해야 합니다.
3. 반응형 디자인: 다양한 기기(데스크톱, 태블릿, 모바일)에서 최적의 사용자 경험을 제공해야 합니다.
4. 관리자 페이지: 포트폴리오 콘텐츠를 쉽게 업데이트할 수 있는 관리자 인터페이스를 제공해야 합니다.

## 추가 요구 사항

- 비개발자인 사용자도 쉽게 콘텐츠를 추가하고 수정할 수 있어야 합니다.
- 관리자 페이지 접근 권한은 인증된 사용자의 구글 계정으로 제한되어야 합니다. 그 외에는 404 페이지로 리디렉션되어야 합니다.
- 페이지 로딩 속도가 3초 이내여야 합니다.
- 웹 사이트는 WCAG 2.1 AA 표준을 준수해야 합니다.
- 주요 웹 브라우저(Chrome, Safari, Edge) 및 모바일 브라우저에서 정상적으로 작동해야 합니다.
- 코드베이스는 잘 구조화되고 문서화되어야 하며, 향후 기능 추가 및 수정이 용이해야 합니다.

**Q1. 사이트 구성 — 페이지를 어떻게 나눌까요?**

방문자가 사이트를 돌아다니는 방식을 정합니다.

| 옵션 | 설명 |
| - | - |
| A. 한 페이지에 모두 표시 (추천) | 스크롤하면 자기소개, 대표 콘텐츠, 블로그가 차례로 나옵니다. 참고 사이트와 비슷한 방식입니다. 콘텐츠가 적을 때 깔끔합니다. |
| B. 메뉴별 개별 페이지 | "소개", "다큐멘터리", "책", "기사" 등 메뉴를 클릭하면 각각 별도 페이지로 이동합니다. 콘텐츠가 많을 때 적합합니다. |
| C. 혼합 | 홈 페이지에는 요약만 보여주고, 블로그 글 목록은 별도 페이지로 나눕니다. |

| B. 대표 콘텐츠 (추천) | 다큐멘터리, 책, 기사 등 대표 작품을 카드 형태로 보여줍니다 |
| C. 최신 블로그 글 (추천) | 블로그에서 최근 작성한 글 몇 개를 미리보기로 보여줍니다 |

| 옵션 | 설명 |

### 1단계: 기술 스택 확정

| 영역 | 기술 | 선택 이유 |
| - | - | - |
| 프레임워크 | Next.js (App Router) + TypeScript | SSG/SSR로 로딩 3초 요건 충족, Vercel 무료 배포 최적 호환, 접근성 생태계 풍부 |
| 스타일링 | Tailwind CSS | 다크 테마 기본 지원, 반응형 유틸리티 내장 |
| 데이터베이스 | Supabase (PostgreSQL) | 무료 티어(DB 500MB, Storage 1GB), 구글 인증 내장 |
| 인증 | Supabase Auth (Google OAuth) | 구글 계정 1개 제한 요건 충족, 별도 서버 불필요 |
| 편집기 | Tiptap (WYSIWYG) | 오픈소스, 이미지/동영상 삽입 지원, 비개발자 친화적 |
| 이미지 저장 | Supabase Storage | 썸네일 수준 이미지, 업로드 시 자동 리사이징 처리, 1GB 무료 |
| 영상 | YouTube 임베드 | 별도 저장 불필요, 유튜브 링크로 대체 |
| 배포 | Vercel | Next.js 공식 호스팅, 무료 도메인(.vercel.app) 제공 |

> 월 운영 비용: 0원 (모든 서비스 무료 티어 내)

### 2단계: 디자인 확정

디자인 A(미니멀 카드 그리드) + 디자인 B(히어로 풀스크린)의 리스트형 블로그를 결합.

- 테마: 어두운 배경, 밝은 텍스트
- 헤더: 이름/소개글 + 메뉴 링크 (미니멀)
- 자기소개: 이름, 직업, 간단한 소개글
- 블로그: 카테고리 탭 필터(다큐멘터리/책/기사) + 리스트형(썸네일 + 제목/요약)
- 푸터: 연락처 (이메일, SNS)
